Har tag ka ek maqsad hota hai. Jab aap inhein sahi tarike se use karte hain, toh aapki website Fast, SEO-friendly, aur Accessible banti hai.

1. Headings: <h1> to <h6>

Headings browser ko batati hain ki content ki hierarchy (importance) kya hai.

Logic: <h1> sabse bada (font-size) aur sabse important hota hai. Jaise-jaise number badhta hai (h2, h3...), size chhota hota jata hai.

Best Practice: Ek page par sirf ek <h1> rakhein. Ye aapki website ka main title hota hai.

H1: Web Development Course

H2: Chapter 2 Name

H3: Topic Name

H4: Details

H5: More Details
H6: Smallest Note

2. Paragraphs: <p> Tag

Ye tag content ko "Blocks" mein todne ke liye use hota hai. Browser paragraph ke upar aur niche automatically Margin (space) add kar deta hai.

‚ö†Ô∏è Whitespace Collapse Rule:

Agar aap <p> ke andar 10 spaces denge, toh browser use sirf 1 space dikhayega.

<p> Ye ek normal paragraph hai. HTML extra spaces ko ignore kar deta hai. </p>

3. Line Breaks: <br> Tag

Jab aapko naya block (paragraph) nahi banana, sirf cursor ko agali line par bhejna ho.

  • Self-Closing: Iska koi closing tag nahi hota.
  • Use Case: Addresses, Poems, ya Lyrics likhne ke liye best hai.
<p>
  Ashish Kumar<br>
  Mau, Uttar Pradesh<br>
  India - 275101
</p>

4. Horizontal Rules: <hr> Tag

Ye ek horizontal line khinch deta hai jo content ko visually alag karti hai. Ye "Thematic Break" (topic change) ko darshata hai.

Example:

(Upar wali line <hr> hai)

5. Centering Content: <center> Tag

Caution (Savdhan)!

Ye tag Deprecated hai. Iska matlab naye browsers ise support karna band kar sakte hain.

Modern Way: CSS ka text-align: center; use karein.

6. Preserving Formatting: <pre> Tag

Agar aap chahte hain ki aapka code, spaces, aur tabs exactly waise hi dikhein jaise aapne editor mein likhe hain, toh <pre> use karein.

Function Hello() {
    print("Hello World");
    //   Spaces and
    //   Indentations
    //   Are Preserved!
}
